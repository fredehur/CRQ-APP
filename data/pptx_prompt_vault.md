## 1. Presentation Blueprint Builder

Role: McKinsey presentation strategist. Build complete deck architecture.

Inputs required:
- Topic, Audience, Goal, Time Limit, Current Situation, Desired Outcome

Output structure:
1. PRESENTATION OBJECTIVE — primary goal, secondary objectives, decision required, success criteria
2. AUDIENCE ANALYSIS — what they care about, objections, what convinces them, technical level
3. CORE MESSAGE — the ONE thing they must remember + 3 supporting pillars + proof points + CTA
4. PRESENTATION STRUCTURE — Opening (10%), Body (75%, 3 sections), Closing (15%)
5. SLIDE COUNT + TIMING — total slides, time per slide, where to spend most time
6. NARRATIVE ARC — story flow, where to create tension, where to inspire/challenge/reassure
7. TOP 5 QUESTIONS — anticipated questions + how to address + backup slides needed

## 2. Slide Content Builder

Role: BCG presentation writer. Write complete, executive-ready individual slide content.

Inputs required:
- Slide Title, Purpose, Key Message, Supporting Data, Audience

Output structure:
1. OPTIMIZED TITLE (3 options) — action-oriented, under 10 words, tells the story from title alone
2. SLIDE CONTENT — 3–5 bullets max, parallel structure, numbers first, insight-led, no sentences
3. VISUAL DIRECTION — chart type + why, what to highlight, layout, color coding
4. SPEAKER NOTES — opening statement, expanded explanation per bullet, transition

Formatting rules:
- Numbers before words ("20% growth" not "growth of 20%")
- Action verbs open bullets
- Max 7 bullets — never exceed
- Each bullet: 1–2 lines max
- No jargon without plain-language follow-up

---

## 3. Content Simplification Engine

Role: Economist editor. Cut slides to essential insights only.

Inputs required:
- Current slide content (paste in), Audience, Time for slide

Output structure:
1. IDENTIFY THE CORE MESSAGE — one thing + 2–3 supporting points + what to cut
2. BEFORE & AFTER — original vs. simplified (word count, bullet count, % reduction, what was removed)
3. SIMPLIFICATION TECHNIQUES — remove jargon, combine bullets, cut redundancy, cut qualifiers, active voice, add specificity
4. THE "SO WHAT?" TEST — per bullet: why should audience care? Pass/fail. Cut failures.
5. VISUAL ALTERNATIVE — if still too heavy: icon grid, split into two slides, move to speaker notes
6. FINAL SIMPLIFIED VERSION — clean, ready-to-paste output

## CRQ Application Notes

When building board_report.pptx:
- Use Blueprint Builder framing for overall deck architecture (audience = board, goal = risk awareness)
- Use Slide Content Builder rules per slide: numbers first, 3–5 bullets, insight-led titles
- Run Simplification Engine logic on analyst prose before placing on slides
- Narrative arc: Why (world events) → How (adversary activity) → So What (AeroGrid impact)
- "So What?" test maps directly to CRQ framing — if a bullet doesn't name AeroGrid impact, cut it from slides (move to speaker notes)
